# 功能建议文档

## 一、当前功能概览

### 用户系统
- **注册/登录**：支持用户名+邮箱注册，用户名或邮箱登录，JWT + HttpOnly Cookie 认证
- **角色权限**：admin / user 两种角色，middleware 路由保护
- **账户状态**：支持启用/禁用用户

### 聊天功能
- **AI 聊天**：基于阿里云百炼平台 Qwen 模型，LangChain 集成
- **流式响应**：ReadableStream 实现实时流式输出
- **会话管理**：新建对话、切换会话、历史会话列表
- **消息持久化**：用户消息和 AI 回复保存到 PostgreSQL
- **Token 额度**：非 admin 用户每日 Token 用量限制，自动按天重置

### RAG 知识库
- **文档上传**：支持 .txt / .md 格式，拖拽上传，最大 2MB
- **向量检索**：pgvector + DashScope text-embedding-v3，HNSW 索引
- **自动 RAG**：聊天时自动检索知识库相关内容作为上下文
- **文档管理**：查看文档列表、状态、删除文档

### 管理后台
- **用户管理**：用户列表、搜索、角色修改、状态切换、新增用户
- **Token 管理**：查看/修改用户额度、重置已用量
- **知识库文档**：查看所有文档及上传者信息
- **统计卡片**：总用户数、管理员数量、文档统计

### UI/UX
- **科幻主题**：粒子背景、扫描线、霓虹发光、脉冲动画等特效
- **暗色模式**：CSS prefers-color-scheme 自动切换
- **响应式设计**：移动端侧边栏折叠、汉堡菜单

## 二、功能建议（按优先级排序）

### 高优先级

#### 1. 会话删除与重命名

- **功能描述**：用户可以删除不需要的历史会话，以及修改会话标题（当前标题由首条消息前 30 字自动生成，无法修改）。
- **用户价值**：让用户更好地组织和管理聊天记录，提升使用体验。
- **涉及改动范围**：
  - 前端：`app/chat/page.tsx` 侧边栏会话项增加右键菜单或操作按钮
  - 后端：`app/api/chat/history/[sessionId]/route.ts` 增加 DELETE 和 PATCH 方法
- **实现复杂度**：简单

#### 2. Markdown 渲染与代码高亮

- **功能描述**：AI 回复支持 Markdown 格式渲染（标题、列表、表格、粗体等），代码块支持语法高亮。当前消息仅以纯文本 + 链接解析展示。
- **用户价值**：AI 回复经常包含代码和格式化内容，Markdown 渲染能极大提升阅读体验。
- **涉及改动范围**：
  - 前端：`app/chat/page.tsx` 消息渲染部分，引入 react-markdown + rehype-highlight 或 prism.js
  - 依赖：新增 react-markdown、rehype-highlight 等包
- **实现复杂度**：简单

#### 3. 消息复制功能

- **功能描述**：每条 AI 回复下方增加"复制"按钮，一键复制消息内容到剪贴板。
- **用户价值**：方便用户复制 AI 生成的代码、文案等内容，是聊天类产品的基础功能。
- **涉及改动范围**：
  - 前端：`app/chat/page.tsx` 消息气泡组件增加复制按钮
- **实现复杂度**：简单

#### 4. 管理后台数据统计仪表盘

- **功能描述**：在管理后台新增"数据统计"页面，展示：用户增长趋势、每日活跃用户数、Token 消耗趋势图、每日消息量统计。
- **用户价值**：管理员能直观了解平台使用情况，为运营决策提供数据支撑。
- **涉及改动范围**：
  - 新增页面：`app/admin/stats/page.tsx`
  - 新增 API：`app/api/admin/stats/route.ts`（聚合查询数据库）
  - 管理后台导航：`app/admin/layout.tsx` 增加导航项
  - 依赖：可引入轻量图表库（如 recharts）
- **实现复杂度**：中等

### 中优先级

#### 5. 对话导出

- **功能描述**：用户可以将某个聊天会话导出为 Markdown 或 TXT 文件，方便保存和分享。
- **用户价值**：方便用户将有价值的对话内容保存到本地，或分享给他人。
- **涉及改动范围**：
  - 前端：`app/chat/page.tsx` 增加导出按钮和逻辑（纯前端生成文件下载）
- **实现复杂度**：简单

#### 6. 多模型切换

- **功能描述**：用户在聊天时可以选择不同的 AI 模型（如 qwen-plus、qwen-turbo、qwen-max），管理员可配置可用模型列表。
- **用户价值**：不同场景可选择不同模型——快速问答用 turbo（更快更省 token），复杂任务用 max（更智能）。
- **涉及改动范围**：
  - 前端：`app/chat/page.tsx` 增加模型选择器
  - 后端：`app/api/chat/route.ts` 接收并使用用户选择的模型
  - 可选：管理后台配置可用模型列表
- **实现复杂度**：中等

#### 7. 系统提示词模板

- **功能描述**：管理员可预设多个系统提示词模板（如"客服助手"、"代码助手"、"翻译助手"），用户新建对话时可选择模板。
- **用户价值**：预设角色让 AI 在特定场景下表现更好，降低用户使用门槛。
- **涉及改动范围**：
  - 数据库：新增 `prompt_templates` 表
  - 后端：CRUD API for 模板管理
  - 前端：新建对话时的模板选择 UI
  - 后端：`app/api/chat/route.ts` 在消息头部插入所选模板的 SystemMessage
- **实现复杂度**：中等

#### 8. 消息搜索

- **功能描述**：用户可以在聊天记录中搜索关键词，快速定位到包含特定内容的会话和消息。
- **用户价值**：积累大量对话后，搜索功能帮助用户快速找到之前的信息。
- **涉及改动范围**：
  - 后端：`app/api/chat/search/route.ts` 新增全文搜索 API
  - 前端：`app/chat/page.tsx` 侧边栏增加搜索框和搜索结果展示
  - 数据库：可选增加 GIN 索引加速全文搜索
- **实现复杂度**：中等

### 低优先级

#### 9. 错误重试机制

- **功能描述**：当 AI 回复失败（网络错误、模型超时等）时，消息气泡显示错误状态和"重试"按钮，用户可一键重新发送。
- **用户价值**：避免用户手动重新输入消息，提升容错体验。
- **涉及改动范围**：
  - 前端：`app/chat/page.tsx` 消息组件增加错误状态和重试逻辑
- **实现复杂度**：简单

#### 10. 知识库支持更多文件格式

- **功能描述**：知识库上传支持 PDF、DOCX、CSV 等更多文件格式，扩大可导入的知识范围。
- **用户价值**：企业知识通常以多种格式存在，支持更多格式降低知识导入成本。
- **涉及改动范围**：
  - 后端：`app/api/kb/upload/route.ts` 增加文件解析器
  - 依赖：引入 pdf-parse、mammoth 等解析库
  - 前端：`app/kb/page.tsx` 更新接受的文件类型
- **实现复杂度**：中等

#### 11. 暗色/亮色模式手动切换

- **功能描述**：当前主题完全跟随系统偏好，增加手动切换按钮，让用户可以自由选择亮色或暗色模式。
- **用户价值**：部分用户希望在特定场景下手动控制主题，不完全依赖系统设置。
- **涉及改动范围**：
  - 前端：新增 ThemeContext 或使用 localStorage 存储偏好
  - 前端：顶部导航栏增加主题切换按钮
  - CSS：`app/globals.css` 调整暗色模式的触发方式（class-based 替代 media query）
- **实现复杂度**：简单

#### 12. 用户个人设置页面

- **功能描述**：用户可查看自己的账户信息、修改密码、查看 Token 使用情况。
- **用户价值**：基本的账户自管理功能，减轻管理员负担。
- **涉及改动范围**：
  - 新增页面：`app/settings/page.tsx`
  - 新增 API：`app/api/auth/password/route.ts`（修改密码）
  - 前端：导航栏增加设置入口
- **实现复杂度**：简单
